Actúa como un desarrollador full-stack senior especializado en UX conversacional, JavaScript y prototipado de agentes con LLM.

Necesito que construyas una aplicación web funcional (MVP) para un **“Agente de Solicitud de Préstamos”** en una **web app** con landing page + formulario de solicitud + integración con modelo de lenguaje + pantalla de confirmación.

# OBJETIVO DEL MVP
Crear una experiencia web integrada donde:
1) El usuario aterriza en una landing page atractiva.
2) Inicia una solicitud de préstamo en un formulario progresivo de 4 pasos.
3) La app envía los datos al modelo de lenguaje (GPT-4o mini) con el contexto del agente definido.
4) La respuesta del modelo se muestra como una sugerencia humana, cálida, clara y accionable.
5) El flujo finaliza con una pantalla de confirmación de solicitud.

⚠️ IMPORTANTE:
- Debes **mantener intacto** el problema, rol, límites y tipo de respuestas del agente de Sprint 1.
- NO conviertas la experiencia en un formulario rígido; cuando el agente haga preguntas, debe sentirse como conversación casual.
- El resultado debe sonar natural, no “texto generado por IA”.
- La app debe ser un prototipo funcional, clara para demo en Replit.

---

# CONTEXTO DEL AGENTE (SPRINT 1) — NO MODIFICAR
Usa este contexto como base para el comportamiento del modelo:

## Problema que resuelve
Reducir fricción en onboarding de créditos, disminuir abandonos y errores, y lograr expediente listo para evaluación con mínima intervención humana.

## Rol del agente
Asistente de onboarding de créditos que guía paso a paso, responde dudas, indica información/documentos, valida datos y prepara precalificación digital en casos de bajo riesgo.

## Tono / personalidad
- Cercano, empático, profesional.
- Claro para clientes; más técnico si el usuario es ejecutivo.
- Respuestas breves a medias.
- Puede usar pasos numerados, checklists y resúmenes estructurados.

## Tipo de respuestas esperadas
- Guía paso a paso
- Checklist/documentos personalizados
- Observaciones claras (si falta algo)
- Estado de solicitud (registro / observada / en evaluación / aprobada / rechazada)
- Precalificación referencial (sin prometer aprobación)

## Límites (obligatorios)
- No prometer aprobación del préstamo
- No tomar decisiones fuera de política crediticia
- No dar asesoría legal/tributaria/inversión
- Cumplir KYC/AML
- Declarar limitaciones y pedir aclaraciones si falta información

## Estilo de UX conversacional
- Mensajes breves, cálidos, claros
- Sensación de acompañamiento y confianza
- No sonar robótico ni excesivamente formal
- Explicar próximos pasos de forma accionable

# MODELO DE LENGUAJE
Usar **gpt-4o-mini**.

Implementa una función de backend o mock server endpoint (por ejemplo `/api/loan-agent`) que:
1. Reciba los datos del formulario y contexto del flujo.
2. Arme un prompt estructurado para el modelo.
3. Llame al modelo.
4. Devuelva una respuesta textual natural + sugerencias accionables.

Si no se puede usar API real por credenciales, deja:
- Integración preparada con variables de entorno (`OPENAI_API_KEY`)
- Un modo fallback/mock con respuesta simulada realista para demo

# EXPERIENCIA DEL AGENTE (INTEGRACIÓN CON LLM)
Después de completar el formulario (o en un panel lateral al final del paso 4), mostrar una tarjeta / bloque conversacional del agente que:

1. Reciba el payload del formulario
2. Lo envíe al endpoint `/api/loan-agent`
3. Muestre una respuesta del agente en tono humano, cálido y accionable

## Qué debe responder el agente (formato esperado)
La respuesta debe incluir, según aplique:
- Saludo breve personalizado
- Resumen corto de la solicitud
- Próximos pasos
- Checklist de documentos sugeridos (si corresponde)
- Observaciones importantes (si hay datos incompletos o riesgos)
- Aclaración de que es una orientación/precalificación referencial (sin prometer aprobación)

## Estilo de la respuesta (muy importante)
- Natural, humana, clara, cálida
- Que parezca sugerencia de un asesor
- No sonar a plantilla rígida
- Sin frases tipo “como IA…”
- En español (Perú), lenguaje claro y profesional

# PROMPT INTERNO PARA EL MODELO (IMPLEMENTAR EN CÓDIGO)
Crea el prompt que se enviará al modelo usando:
- Un **system prompt** (rol del agente y límites)
- Un **user prompt** (datos del formulario)

## Requisitos del system prompt
Debe incluir:
- Rol del agente (onboarding de créditos)
- Problema que resuelve
- Tono/persona
- Límites (no prometer aprobación, KYC/AML, etc.)
- Tipo de salida esperado (orientación clara, próxima acción, checklist si aplica)
- Instrucción explícita para sonar natural y humano
- Instrucción para preguntar de forma conversacional si faltan datos críticos (no interrogatorio)

## Requisitos del user prompt
Debe pasar datos estructurados del formulario como JSON legible, por ejemplo:
- tipoUsuario
- tipoProducto
- tipoDocumento
- nroDocumento
- nombre
- email
- telefono
- monto
- moneda
- plazoMeses
- tipoPago
- ingresosMensuales
- actividadEconomica
- empresaORubro
- antiguedadLaboral
- aceptaTyC

También puedes incluir:
- `stage: "post_form_submission"`
- `canal: "web_app"`

## Formato de salida recomendado del modelo (texto)
Respuesta natural en español con secciones suaves (sin rigidez excesiva), por ejemplo:
- breve saludo
- resumen
- próximos pasos
- documentos sugeridos
- observaciones
- cierre empático

No mostrar JSON al usuario final (solo texto).
